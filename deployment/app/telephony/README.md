# Telnyx Phone Integration for SmartShip

This directory contains the telephony integration layer for connecting SmartShip Voice Assistant to phone networks via Telnyx.

## 📁 Directory Structure

```
telephony/
├── __init__.py           # Package exports
├── audio_transcode.py    # Audio format conversion (μ-law ↔ PCM16)
├── telnyx_handler.py     # Main WebSocket handler for phone calls
└── README.md            # This file
```

## 🎯 What This Integration Does

Enables users to call SmartShip Voice Assistant from any phone:
- **Regular phone calls** (landline or mobile)
- **Audio-only interaction** (no camera/video)
- **Uses IVR mode** with intelligent conversation flow
- **Real-time bidirectional audio** streaming

## 🔧 Key Components

### 1. `telnyx_handler.py`
Main WebSocket handler for Telnyx phone calls:
- Accepts WebSocket connections from Telnyx
- Manages bidirectional audio streaming
- Bridges phone network with Gemini ADK agent
- Handles session lifecycle

### 2. `audio_transcode.py`
Audio format conversion utilities:
- **Incoming**: μ-law 8kHz → PCM16 16kHz (for Gemini)
- **Outgoing**: PCM16 24kHz → μ-law 8kHz (for phone)
- Sample rate conversion
- Audio chunking for streaming

## 📡 WebSocket Protocol

### Telnyx → Server Messages

```json
// Call started
{
  "event": "start",
  "call_control_id": "v3:xxx",
  "call_leg_id": "xxx"

// Audio data from caller
{

  "payload": "base64_encoded_mulaw_audio"

```

### Server → Telnyx Messages

```json
// Send audio to caller
{
  "event": "media",
  "payload": "base64_encoded_mulaw_audio"
}
```

## ⚙️ Configuration

### Environment Variables
```bash
GOOGLE_API_KEY=your_gemini_api_key  # Required for ADK
```

### Telnyx Setup

1. **Get a phone number** from [Telnyx Portal](https://portal.telnyx.com)
2. **Configure TeXML Application**:
   - Go to Numbers → My Numbers
   - Select your number → Voice Settings
   - Connection Type: TeXML Application
   - Webhook URL: `https://your-domain.com/texml/voice`

3. **TeXML Response** (auto-generated by `/texml/voice` endpoint):
```xml
<?xml version="1.0" encoding="UTF-8"?>
<Response>
    <Say voice="alice">Connecting you to Smart Ship Assistant</Say>
    <Stream url="wss://your-domain.com/ws/phone" />
</Response>
```

## 🔄 Audio Flow

### Incoming (Caller → Gemini)
1. Telnyx sends μ-law audio @ 8kHz (phone network standard)
2. Handler decodes base64 payload
3. Transcode to PCM16 @ 16kHz
4. Send to Gemini ADK agent

### Outgoing (Gemini → Caller)
1. Gemini generates PCM16 audio @ 24kHz
2. Transcode to PCM16 @ 8kHz (downsample)
3. Convert to μ-law format
4. Base64 encode and send to Telnyx

## 🎭 Session Management

Each phone call gets its own:
- **User ID**: `phone_{call_control_id}`
- **Session ID**: `{call_control_id}`
- **ADK Runner**: Separate instance per call
- **Agent Mode**: IVR (audio-only, uses `system_prompt_ivr.txt`)

Sessions are automatically cleaned up when call ends.

## 🚀 Production Considerations

- **WebSocket timeout**: Ensure server supports long-lived connections (3600s+)
- **HTTPS required**: Telnyx requires secure WebSocket (wss://)
- **Concurrency**: Each call runs in its own async task
- **Resource cleanup**: Sessions properly cleaned up on disconnect
- **Error handling**: Graceful degradation on transcoding/connection errors
